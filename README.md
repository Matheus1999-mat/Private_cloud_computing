# Private_cloud_computing

## English

## About the project
Design of a private cloud computing platform focused on storage, synchronization and file collaboration. It is targeted at environments that require full control over data and infrastructure, without relying exclusively on public cloud providers. The solution prioritizes data sovereignty, security and autonomy.

## Objectives

- **To offer a private cloud alternative for internal use.**
- **To centralize file storage and collaboration within a controlled environment.**
- **To reduce risks associated with the exposure of sensitive data.**
- **To demonstrate an architecture for secure and sustainable infrastructure.**

## Solved problems

- **Exposure of sensitive data to unauthorized individuals.**
- **Excessive dependence on public cloud services.**
- **Risks of unavailability due to external failures.**
- **Difficulty in auditing and maintaining control.**

## Lessons learned

- **An isolated lab environment is essential for testing without compromising real environments.**
- **Deep understanding of Bash, Linux, cloud computing, infrastructure and web servers.**
- **Cloud computing configuration requires careful attention to security rules and user needs.**
- **To seek alternatives is essential to avoid external failures.**


## Português

## Sobre o projeto
Projeto de uma plataforma de computação em nuvem privada focada em armazenamento, sincronização e colaboração de arquivos. Voltado para ambientes que requerem controle total sobre dados e infraestrutura, sem depender exclusivamente de provedores de nuvem pública. A solução prioriza soberania de dados, segurança e autonomia.

## Objetivos

- **Oferecer uma alternativa privada para uso interno.**
- **Centralizar armazenamento de arquivos e colaboração dentro de um ambiente controlado.**
- **Reduzir riscos associados à exposição de dados sensíveis.**
- **Demonstrar uma arquitetura para infraestrutura segura e sustentável.**

## Problemas resolvidos

- **Exposição de dados sensíveis a indivíduos não autorizados.**
- **Dependência excessiva de serviços de nuvem pública.**
- **Riscos de indisponibilidade devido a falhas externas.**
- **Dificuldade de auditoria e manutenção de controle.**

- **Buscar alternativas é essencial para evitar falhas externas.**
